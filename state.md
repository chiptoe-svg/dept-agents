## Goal

## Invariants (don't break)

These are load-bearing. If a commit would violate one, surface it before writing the code.

- **The two-DB session model is the only IO surface between host and container.** `inbound.db` is host-write / container-read. `outbound.db` is container-write / host-read. There is no IPC, no stdin piping, no file-watch. See `container/agent-runner/src/db/connection.ts` and `docs/db-session.md`.
- **`outbound.db` is container-owned; the host opens it readonly except for `host-sweep.ts`'s claim cleanup and `session-manager.ts`'s `writeOutboundDirect` (RW opener).** Any new host-side write to outbound.db requires using `openOutboundDbRw` AND must be crash-safe (stale `-journal` files from a SIGKILL leave the file in a "needs recovery" state that breaks subsequent readonly reads ‚Äî `recoverStaleOutboundJournals` runs at startup to clean them).
- **Session DBs use `journal_mode=DELETE`, not WAL.** Cross-mount visibility depends on this. The pragma is in `container/agent-runner/src/db/connection.ts` and in the host openers in `src/db/session-db.ts`.
- **Container-side bun:sqlite needs `$name` in JS keys** (`db.prepare(...).run({ $id: foo })`); host-side better-sqlite3 auto-strips. Mixing the conventions silently breaks named-param SQL.
- **Trunk is infrastructure-only.** Channel adapters (Discord, Slack, Telegram, ‚Ä¶) live on the `channels` branch; non-default agent providers live on the `providers` branch. Each is installed by a skill. Don't add channel-specific or provider-specific code to trunk. See `CLAUDE.md` ¬ß 5.
- **pnpm `minimumReleaseAge: 4320` (3 days) is enforced.** New deps wait 3 days from publish. `minimumReleaseAgeExclude` and `onlyBuiltDependencies` additions require explicit human approval ‚Äî never bypass.
- **Credentials never sit in container env vars or chat context.** Containers get placeholder credentials; the host credential-proxy at `127.0.0.1:3001` substitutes real keys/tokens at request time. The 4 Anthropic-OAuth landmines (sk-ant-oat- prefix, `anthropic-beta: oauth-2025-04-20`, Claude Code preamble, model.baseUrl override) are all live and must be preserved. See `src/credential-proxy.ts`.
- **`agent_groups.agent_provider`, `container_configs.provider`, and `sessions.agent_provider` must all be `'pi'` post-Phase-D.** Migration 023 backfilled this; `scripts/migrate-to-pi.ts` is the recovery tool. Drift between the three columns triggers the wrong auth-dir mount and was the bug `migrate-to-pi.ts` was written to fix.
- **MCP tool handlers (`container/agent-runner/src/mcp-tools/`) run in a SEPARATE PROCESS from the poll-loop.** Module-level state in `current-batch.ts` is NOT shared. Anything needing to flow from poll-loop to MCP tools must go through the DB (or the spawned tool's args), not module globals. The messages_out usage backfill (commit `b204567`) uses a seq lower bound for exactly this reason.
- **Container `--rm` flag means container stdout/stderr is lost on exit.** If an agent silently failed inside the container, there's no persistent log. Make session-DB writes the source of truth for anything diagnosed post-hoc.

**2026-07-10 ‚Äî The bookmarkable login-token URL is the department's auth credential; bypass retired (Plan 3).** The durable per-user token (`class_login_tokens`, `GET /?token=` ‚Üí session) already existed and is generic ‚Äî Plan 3 provisions users into it (`ncl users provision`) rather than building new auth. **2026-07-10 ‚Äî Never hardcode the container bridge gateway (Plan 2, Apple Container 1.1.0).** The 1.1.0 upgrade moved the bridge subnet `192.168.64.0/24` ‚Üí `192.168.65.0/24` (undocumented), and `bridge100` exists only while a container runs. The hardcoded `192.168.64.1` fallback would have handed every container a dead proxy URL ‚Äî a silent, total outage. The gateway is now resolved at runtime (`container network inspect default` ‚Üí interface scan ‚Üí **throw**, never a constant) in four places on this box: both nanoclaw installs, the CUassistant TCP bridge, and gc_alumni's FastMCP allow-list. *Why it recurs:* a wrong-but-plausible constant turns a documented network change into an invisible failure. Also learned: `container ls --format json` `status` became `{state}` (was a bare string) ‚Äî anything matching `status === 'running'` silently matches nothing. See memory `reference-container-1.1.0-gateway`.

**2026-07-10 ‚Äî Budget enforcement belongs at the credential proxy, not per-route (Plan 2).** Budgets previously enforced only on `/api/direct-chat`; the main agent-turn path (container ‚Üí proxy ‚Üí OpenAI) had no cap and `assertWithinBudget` no-opped with no budget set. Enforcement moved to the proxy ‚Äî the one chokepoint every LLM call crosses, which already knows the token-derived group and cannot be bypassed by a compromised container. `defaultMonthlyUsd=25` now applies to every group including the owner's. *Corollary:* free/local routes (`omlx`) are exempt (blocking them protects $0 and removes the fallback), campus routes of unknown billing (`clemson`) fail closed, and a corrupt `cost-budgets.json` fails closed rather than silently disabling all caps.

**2026-07-10 ‚Äî Identity is derived server-side, never asserted by the caller (Plan 1.5).** A full code review found one defect repeated in three subsystems: the web API took its target agent group from the URL, and the credential proxy and GWS relay took theirs from an `x-nanoclaw-agent-group` header the container itself set. All three now derive identity from something the caller cannot forge ‚Äî the session for routes, a per-container token minted at spawn for the two host services ‚Äî and `ncl` reads are scoped to the host-stamped caller group. *Why:* the department server gives ~15 colleagues their own agents; "one user's secrets, conversations, and results stay separate unless they share them" was the product's core promise and was not actually enforced. Corollary rules for future work: **never authorize from a value the caller supplied**, and `checkDraftMutation` is a class-lockdown hook that default-allows ‚Äî it is not authorization. **Retiring `PLAYGROUND_AUTH_BYPASS` is blocked on Plan 3**, because the seat picker only exists in bypass mode and the magic-link path needs a roster the dept server doesn't have yet.
